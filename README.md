# 💒 婚礼宾客统计系统

婚礼宾客名单管理本地小工具：会场比例平面图、拖拽换桌、邀请函状态记录、人数预算与确认统计、宾客名单 Excel 导入导出。韩式婚礼配色。

- 数据存 `data/wedding.db`（SQLite）；宾客名单通过 **Excel 导入/导出** 与外部交换
- 旧版本的 `guests.xlsx` / `tables.csv` / `config.csv` 数据文件在首次启动时自动迁移入库（原文件保留不动）
- 后端 Python（FastAPI + openpyxl），前端零依赖单页 HTML

## 快速开始

```bash
./run.sh            # 默认 8321 端口，首次运行自动安装依赖、生成示例数据
./run.sh 9000       # 指定端口
```

打开 <http://127.0.0.1:8321/> 即可使用。

## 功能

| 页签 | 说明 |
|---|---|
| 🪑 桌位图 | **会场按比例平面图**（长宽可配，默认 18 × 25 米，1 米背景网格）：顶部舞台 + 中央 T 台走道，圆桌按实际直径（默认 1.8 米）等比绘制，默认适应窗口、可缩放 50%–250%。未手动摆放的桌**按桌号从左到右自动排列**（避开舞台/T台），「🔄 重新排列」可一键恢复有序布局；**拖动圆桌摆放位置**（自动保存坐标）；**点击圆桌**查看单桌详情（宾客列表、换桌、编辑）；把右侧「未安排宾客」名牌**拖到圆桌上即可入座**；新增桌子时桌号自动递增预填 |
| 📋 宾客名单 | 宾客增删改查、搜索；邀请函状态 / 确认状态支持行内下拉直接修改；每行提供「换桌」入口；**导出 / 导入 Excel**（见下） |
| 🛠 桌子管理 | 表格视角管理所有桌子，容量留空表示使用全局默认容量 |
| ⚙️ 设置 | 默认单桌容纳人数、人数预算、会场宽/长（米）、桌子直径（米） |

### 核心概念

- **宾客**：一条记录 = 一组同行的人（本人 + 家属），可配置**预算人数**（邀请时预计的人数）与**确认人数**（宾客确认会到的人数，留空默认等于预算人数）、家属姓名（逗号分隔）、桌号、邀请函状态（未发送/已发送）、确认状态（待确认/已确认/不参加）、备注。
- **占座口径**：已确认的宾客按「确认人数」占座与计数，待确认按「预算人数」，不参加不占座。
- **换桌（可选是否带家属）**：换桌时若预算人数 > 1，可选择「不带家属」——本人（1 人）换到新桌，其余家属留在原桌，名单自动拆分为两条记录（确认人数同步拆分）。
- **容量控制**：每桌容量 = 单桌配置，未配置则用全局默认。换桌 / 安排宾客超出容量时会拦截提示，可选择强制安排（圆桌标红超员）。
- **统计条**：人数预算、预算合计、预计出席（已确认到场 + 待确认）、已确认到场、待确认、不参加、已安排桌位、未安排、邀请函发送进度。预计出席超预算时标红。

## 数据存储与 Excel 导入导出

数据存 `data/wedding.db`（SQLite，三张表：guests / tables / config），首次启动自动建库；若同目录存在旧版 `guests.xlsx` / `tables.csv` / `config.csv` 会自动迁移入库（原文件保留），否则写入示例数据。

**宾客名单 Excel 交换格式**（导出与导入共用，可先导出获得模板）：

| 姓名 | 配偶 | 子女1 | 子女2 | … | 预计人数 | 确认人数 | 桌号 |
|---|---|---|---|---|---|---|---|

- 家属名平铺为多列：第一位为配偶，其余为子女；导出时按「预计人数 − 1」补足空单元格（空位表示还有几人未登记姓名）
- 导入时：预计人数留空按「本人 + 家属数」推断，且不小于已列出的家属数；确认人数留空默认等于预计人数（超出预计会被夹回）
- 导入名单中出现的新桌号自动建桌；导入宾客的邀请函 / 确认状态为初始值（未发送 / 待确认）
- 导入支持**追加**与**覆盖全部**两种模式；批量导入不做容量拦截，可在平面图上再调整

## API 一览

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/overview` | 总览：宾客、桌子（含占用）、未安排、配置、统计 |
| POST / PUT / DELETE | `/api/guests[/{id}]` | 宾客增改删（`force=true` 可强制超容安排） |
| POST | `/api/guests/{id}/move` | 换桌：`{target_table, with_family, force}`，`target_table=""` 表示移入未安排区 |
| GET | `/api/guests/export` | 导出宾客名单 xlsx |
| POST | `/api/guests/import?mode=append\|replace` | 导入宾客名单（body 为 xlsx 原始字节） |
| POST / PUT / DELETE | `/api/tables[/{桌号}]` | 桌子增改删（重命名桌号会同步宾客；删桌后宾客进未安排区） |
| PUT | `/api/tables/{桌号}/position` | 平面图拖动摆位：`{x, y}`（米，保留 1 位小数） |
| POST | `/api/tables/reset-positions` | 清空全部桌子坐标，恢复按桌号自动排列 |
| PUT | `/api/config` | 全局配置：`{default_capacity, budget_total, venue_width, venue_depth, table_diameter}` |

## 项目结构

```
backend/
  app.py        # FastAPI 路由 + 业务逻辑（换桌拆分、容量校验、统计、导入导出）
  storage.py    # SQLite 存储层（含旧版 xlsx/csv 一次性迁移）
frontend/
  index.html    # 单页应用（原生 JS，无外部依赖）
data/           # 运行时数据 wedding.db（首次启动自动生成）
run.sh          # 一键启动
```
