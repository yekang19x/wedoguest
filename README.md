# 💒 婚礼宾客统计系统

用 Excel 管理婚礼宾客名单的本地小工具：可视化桌位排布、拖拽换桌、邀请函状态记录、人数预算与确认统计。

- **主要数据**（宾客名单）存 `data/guests.xlsx` —— 可随时直接用 Excel 打开编辑，刷新页面即生效
- **次要数据**（桌子属性、全局配置）存 `data/tables.csv`、`data/config.csv`
- 后端 Python（FastAPI + openpyxl），前端零依赖单页 HTML

## 快速开始

```bash
./run.sh            # 默认 8321 端口，首次运行自动安装依赖、生成示例数据
./run.sh 9000       # 指定端口
```

打开 <http://127.0.0.1:8321/> 即可使用。

## 功能

| 页签 | 说明 |
|---|---|
| 🪑 桌位图 | **会场按比例平面图**（长宽可配，默认 18 × 25 米，1 米背景网格）：顶部舞台 + 中央 T 台走道，圆桌按实际直径（默认 1.8 米）等比绘制。**拖动圆桌摆放位置**（自动保存坐标，未摆放的桌自动布局避开 T 台）；**点击圆桌**查看单桌详情（宾客列表、换桌、编辑）；把右侧「未安排宾客」名牌**拖到圆桌上即可入座** |
| 📋 宾客名单 | 宾客增删改查、搜索；邀请函状态 / 确认状态支持行内下拉直接修改；每行提供「换桌」入口 |
| 🛠 桌子管理 | 表格视角管理所有桌子，容量留空表示使用全局默认容量 |
| ⚙️ 设置 | 默认单桌容纳人数、人数预算、会场宽/长（米）、桌子直径（米） |

### 核心概念

- **宾客**：一条记录 = 一组同行的人（本人 + 家属），可配置**预算人数**（邀请时预计的人数）与**确认人数**（宾客确认会到的人数，留空默认等于预算人数）、家属姓名（逗号分隔）、桌号、邀请函状态（未发送/已发送）、确认状态（待确认/已确认/不参加）、备注。
- **占座口径**：已确认的宾客按「确认人数」占座与计数，待确认按「预算人数」，不参加不占座。
- **换桌（可选是否带家属）**：换桌时若预算人数 > 1，可选择「不带家属」——本人（1 人）换到新桌，其余家属留在原桌，名单自动拆分为两条记录（确认人数同步拆分）。
- **容量控制**：每桌容量 = 单桌配置，未配置则用全局默认。换桌 / 安排宾客超出容量时会拦截提示，可选择强制安排（圆桌标红超员）。
- **统计条**：人数预算、预算合计、预计出席（已确认到场 + 待确认）、已确认到场、待确认、不参加、已安排桌位、未安排、邀请函发送进度。预计出席超预算时标红。

## 数据文件

| 文件 | 格式 | 内容 |
|---|---|---|
| `data/guests.xlsx` | Excel | 宾客主数据：ID、姓名、预算人数、确认人数、家属姓名、桌号、邀请函状态、确认状态、备注（首行冻结，可直接用 Excel 维护；旧版「总人数」表头可自动兼容读取） |
| `data/tables.csv` | CSV (UTF-8 BOM) | 桌号、备注名、容纳人数（空 = 用全局默认）、X坐标、Y坐标（桌心位置，米；空 = 未摆放，平面图自动布局） |
| `data/config.csv` | CSV (UTF-8 BOM) | `default_capacity` 默认单桌容量、`budget_total` 人数预算、`venue_width`/`venue_depth` 会场宽长（米）、`table_diameter` 桌子直径（米） |

数据文件不存在时首次启动会自动生成（含示例数据，可全部删除后录入真实名单）。每次请求实时读取文件，因此**在 Excel 中修改保存后，刷新网页即可看到最新数据**（注意：Excel 打开 xlsx 文件期间系统写入会失败，编辑完请关闭 Excel）。

## API 一览

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/overview` | 总览：宾客、桌子（含占用）、未安排、配置、统计 |
| POST / PUT / DELETE | `/api/guests[/{id}]` | 宾客增改删（`force=true` 可强制超容安排） |
| POST | `/api/guests/{id}/move` | 换桌：`{target_table, with_family, force}`，`target_table=""` 表示移入未安排区 |
| POST / PUT / DELETE | `/api/tables[/{桌号}]` | 桌子增改删（重命名桌号会同步宾客；删桌后宾客进未安排区） |
| PUT | `/api/tables/{桌号}/position` | 平面图拖动摆位：`{x, y}`（米，保留 1 位小数） |
| PUT | `/api/config` | 全局配置：`{default_capacity, budget_total, venue_width, venue_depth, table_diameter}` |

## 项目结构

```
backend/
  app.py        # FastAPI 路由 + 业务逻辑（换桌拆分、容量校验、统计）
  storage.py    # xlsx / csv 读写层（原子写入）
frontend/
  index.html    # 单页应用（原生 JS，无外部依赖）
data/           # 运行时数据（首次启动自动生成）
run.sh          # 一键启动
```
